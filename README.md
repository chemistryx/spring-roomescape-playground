- ‘초록 스터디’에서 제공하는 학습 자료와 환경을 바탕으로 진행하는 스터디입니다.

## 스터디의 목표는 무엇인가요?

- “클린 코드”, “테스트 코드”, … 등 유지 보수를 위한 각종 도구들을 사용해보면서 **특정 도구에 대한 자신만의 철학 가지기**
- 웹 애플리케이션 구현을 위해 필요한 새로운 기술을 학습하며, **자신의 상황에 맞는 본인만의 학습 방법**을 찾는 시도하기
- “기능을 구현할 수 있는 사람”이 아닌 “**잘 변할 수 있는 소프트웨어를 만들 수 있는 개발자**”가 되기 위해 노력하기

# 무엇을 얻었나요?

콩콩밥밥 스터디는 일주일간 미션을 진행하고 리뷰어에게 리뷰를 받습니다. 그리고 한 주의 마무리를 `같은 내용으로 스터디를 진행한 사람들이 모여서 네트워크 밋업`을 진행했습니다. 미션을 진행하면서 고민했었던 내용을 공유하고, 비슷한 문제를 겪은 사람들과 소통을 하며 다양한 생각을 공유하였습니다. 처음에는 서먹했지만 이후에는 자유로운 분위기에서 질문하고 답변하는 시간을 가졌습니다. 또한 리뷰어분들에게 어떻게 공부하는게 좋을지도 고민하던 내용을 질문하며 소통하는 시간이 너무 좋았습니다.

저는 이 스터디를 진행하면서 부트캠프를 병행하고 있습니다. 수업도 들어야하고 과제도 진행해야하기 때문에에 스터디에 하루에 2시간만 투자할 수 있었습니다. 주어진 시간 안에서 최선의 아웃풋을 얻기 위해서 투자하는 시간 동안에는 최선을 다해 몰입하는 경험을 했습니다.

### Code

스터디를 통해 코드를 작성할 때 상황에 맞는 근거를 찾고 적절한지에 대해 고민을 하게 되었고, 내가 미션을 해결하기 위해서 자료를 찾아 읽어보면서 다양한 방법과 비교하는 자세를 배웠습니다. 이를 통해 나만의 철학을 만들고 적용해나가고 있습니다.

나의 상황에 맞는 본인만의 학습 방법 -> 현재 하고 있는 일이 많지만 이 상황 속에서 내가 얻어갈 수 있는 최선의 방법을 계속해서 고민했고, 리뷰어들에게 현재 나의 고민들을 공유했습니다다. 나보다 먼저 이런 고민들을 했던 선배들은 어떻게 해결해나갔는지 질문하며 생각의 확장을 할 수 있었습니다.

### 고민하고 상담받고 했던 기록

```text
# 현재 내 상황과 생각을 공유
최근 스터디도 진행해보고 학원에서 수업도 듣고 과제도 하면서 드는 생각이 있어요. 공부하는 방식인데요.
저는 어떤 목표를 세우고 계획을 세우려고 시도했어요. 그런데 생각보다 안해봐서 어렵더라고요.
인터넷에서 다른 사람들의 생각도 읽어보면서 어떻게 하는게 좋을까 고민했어요.

구현하고 싶은 것을 생각해보고, 해당 목표를 쪼개고 쪼개서 하나의 단위로 만들어본다. 그 단위 안에서 학습해야하는 부분을 찾아내고
관련 자료를 학습하고 정리한후 나의 생각을 정리해서 문서화한다. 그리고 관련 실습을 진행하고 나의 프로젝트에 적용해봐야겠다.

하루하루 작은 목표들이 생겨서 덜 불안할 것 같다는 장점이 있고 자신감도 생길것 같아요. 단점으로는 단위를 쪼개고
학습해야하는 부분을 찾아내는 것이 쉽지만은 않은 것 같다, 작은 한 단위 사이클이 길어질 수 있을 것 같아요.

현재 구현하고 싶은 프로젝트를 진행하려 하는데 잘 만들고 잘 설명할 수 있도록 만들어보고 싶어서 질문드렸습니다.

# 리뷰어-커찬
공부 방법
구현하고 싶은 것을 생각해보고, 해당 목표를 쪼개고 쪼개서 하나의 단위로 만들어본다.
그 단위 안에서 학습해야하는 부분을 찾아낸다.
관련 자료를 학습하고 정리한후 나의 생각을 정리해서 문서화한다.
관련 실습을 진행하고 나의 프로젝트에 적용해본다.
- 장점 : 하루하루 작은 목표들이 생겨서 조금 덜 불안하다.
- 단점 : 한 작은 단위 사이클이 길어진다.
목적
내가 구현하는 프로젝트에 사용하는 것을 잘 설명하고 싶고 잘 만들어보고 싶어서

조금 더 생각해봐야 할 점이라면...
잘 설명하는 사람은 어떤 사람인가요? 잘 설명하면 무엇이 좋은가요? (ex. 취업을 잘 할 수 있다. 등등...)
잘 만들어본다는 것은 무엇인가요? 잘 만들면 무엇이 좋은가요? (ex. 수익화를 할 수 있다. 이력서에 쓰기 좋다. 등등...)
```

<br/>

나의 생각을 커찬이 다시 한번 정리해주며 내가 어떤 것을 원하고 어떤 사람이 되고 싶은지 생각할 수 있도록 유도해주어서 구체적으로 어떤 사람이 되고 싶은가에 대해서 생각할 수 있었습니다.

<br/>

**나의 생각을 정리**  
`잘 설명하는 사람` : 구현 방식에 대해서 왜 선택했는지, 고려해본 점은 무엇인지 말로 설명하는 사람. 이렇게 정리할 경우 오래 기억할 것 같아서.

`잘 만들어 본다` : 현재 프로젝트 구상을 하고 있는데, 주제를 설정했어요. 실제 배포해서 사용자에게 제공하고 유지보수까지 하는게 목표에요. 즉 제가 만든 서비스가 지속적으로 유지보수되고 실제 운영하는게 저의 `잘만들어본다` 인 것 같아요.

<br/>

> **커찬 조언**
> 처음부터 맞는 공부 방식은 없으니까 1~2주 간격으로 회고하는 시점을 잡아서 현재 내가 생각한대로 잘 가고 있는지, 더 개선할 방법은 없는지 점검하면서 가면 좋을 것 같아요.

### 가장 기억에 남는 부분
부트캠프가 수업이 끝나고 한 숨 자고 일어나서 해야하는 부분들을 해나가던 중 새벽에 리뷰어와 고민상담을 하게되었다. 오전에 고민을 하고 있던 부분을 디엠으로 남겨두었는데 마침 시간이 맞아서 새벽 늦게까지 음성통화로 소통하게 되었다.

```text
# 나의 고민
기존에 작성한 코드가 미션을 진행하면서 새로운 요구사항을 구현하고 보니 다 깨져서 멘붕이 왔다.
이런 경험이 없었고 어디서부터 수정을 해야하고 언제 다할지에 대해서 생각이 들어서였다.

천천히 하나하나 수정해두었다. 그 과정에서 내 테스트 코드가 깨진것은 기존의 테스트 코드가 
잘 작동하고 있구나라는 생각을 했었다. 그리고 콜리와 통화하면서 이에 대해서 조금 더 깊게
이야기를 나누며 생각의 확장을 했다. 

`테스트 코드가 깨지는게 과연 내가 유지보수하기 힘든 구조로 짯기 때문인가?` 를 생각하게 되었고
변경이 일어날 때 테스트 코드가 꺠져야 정상이다. 그렇지 않다면 제대로 테스트 코드를 짯는가에
대해서 생각해봐야 한다.

아직 촘촘히 코드를 짜는 실력이 부족하고 공통 관심사를 분리할 수 있는 눈이 부족하다. 계속해서
생각하고 시도해보면서 시점을 기르는 부분이 필요하다. 그리고 현재 하고 있는게 학습을 위한
미션이지 프로덕트를 내는 것이 아니니 다양한 시도와 많은 실패를 해보라는 조언을 받았다.

# 현재 하고 있는 프로젝트에 대한 고민 상담
내가 진행하고 있는 프로젝트에서 막히는 부분과 막막한 부분을 상담했다. 문제점에 대해서 인식
하지 못하고 있었는데, 콜리의 도움으로 머리가 뚫리는 느낌을 받았다.

내가 작성한 프로젝트 명세서와 계획서를 공유했고, 현재 내가 프로젝트를 진행하는데 있어
가장 핵심적인 가치, 제공하고자하는 것에 대해서 다시 생각하고 깊게 고민하게 되었다.
어떤 문제점이 있었고, 정확하게 어떻게 해결할 것인지에 대해서 명확하지 않기에 설계와 구현이
명확하지 않을 수 있다라는 것을 알았다.

- 나의 프로젝트는 다이어트가 필요하다.
- 내가 만드는 서비스의 핵심가치는 무엇인가에 대해서 진지하게 고민
- 유저 인터뷰의 필요성

```

이외에도 많은 질문과 답변을 통해서 소통의 중요성을 알게되었습니다. 또한 기술적인 부분이나 개인 고민 등 소통할 수 있는 많은 시간을 가졌습니다.
